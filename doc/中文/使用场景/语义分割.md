# 语义分割推理（semantic）

本篇演示如何用 `semantic` 任务根做 YOLO26 语义分割：对每个像素预测一个类别，输出整张图的稠密类别图（不是实例级的）。这与实例分割（segment）不同——semantic 不区分个体，只区分「这一片像素属于什么类别」。

完整 API 参考见 [任务推理API.md](../任务推理API.md)。

## 何时用

- 逐像素分类任务，如自动驾驶场景理解（Cityscapes）、遥感地物分类。
- 不关心实例边界，只关心区域类别。
- 需要稠密 logits map 自行做 argmax/可视化。

## 准备

- **Feature**：需 `--features semantic`（不在默认 feature 里）。
- **权重**：语义分割官方 `.pt` 权重名形如 `yolo26s-sem.pt`（`-sem` 后缀）。
- **默认类别**：Cityscapes（19 类）。`labels_count` 从 checkpoint head shape 推断；类名表 `default_labels::CITYSCAPES` 需启用 `default_labels` feature（默认关），或用 `.pt` 权重的 `pt_loader::load_pt_metadata().names` 读取。

```bash
cargo build --release
```

## Rust 示例

```rust
use yolo26_rs::{Image, MaskOption, semantic};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let image = Image::from_file("examples/bus.jpg")?;
    let model = semantic::Model::from_file("yolo26s-sem.pt")?;
    let pred = model.predict(&image, &MaskOption { high_resolution: true })?;
    let class_ids = pred.class_ids();
    println!("map={}x{} classes={}", pred.width, pred.height, pred.class_ids().len());
    for y in 0..pred.height as usize {
        for x in 0..pred.width as usize {
            let cid = class_ids[y * pred.width as usize + x];
            let _ = cid;
        }
    }
    Ok(())
}
```

## 对应官方 Python 代码

YOLO26 官方语义分割路径通过专门的语义分割模型/任务接口运行；典型用法：

```python
from ultralytics import YOLO

model = YOLO("yolo26s-sem.pt")
results = model.predict("examples/bus.jpg", imgsz=640)
mask = results[0]
```

> 官方语义分割的精确 result 对象形态请以当前 Ultralytics 文档为准；本 crate 的对齐目标是逐像素稠密 logits。

## API 与配置项详解

`semantic::config_builder()` 默认输入 640、类别数 **Cityscapes**。builder 方法与 detect 一致（见 [物体检测.md](物体检测.md)）；自定义数据集用 `with_labels_count`。

### 模型与预测

| API | 说明 |
| --- | --- |
| `semantic::Model::from_file(path)` | 自动识别 `.pt` / `.safetensors` 并加载语义分割模型。 |
| `semantic::Model::from_pt_file(path, config)` | 从官方 `.pt` 加载。 |
| `semantic::Model::from_safetensors(weights, config)` | 从 `.safetensors` 字节加载。 |
| `model.forward_tensor(&input)` | raw forward，返回 `[B, nc, H, W]` 稠密 logits。 |
| `model.predict(&image, &mask)` | 端到端推理，返回 `semantic::Prediction`。 |

### `semantic::Prediction`

| API | 说明 |
| --- | --- |
| `width` / `height` | 输出图尺寸（受 `MaskOption` 影响）。 |
| `classes` | 类别数 `nc`。 |
| `logits` | 逐像素 logits。 |
| `class_id(x, y)` | 返回该像素 argmax 类别 id。 |
| `class_ids()` | 返回与输出图同尺寸的 `Vec<u32>` 类别图。 |
| `resize(width, height)` | 把 logits map 重采样。 |

### `MaskOption`

语义分割也用 `MaskOption { high_resolution }` 控制输出图是否上采样到源图分辨率（与 segment 一致）。

## 与官方的差异 / 注意事项

- 官方语义分割结果对象的访问形式随版本变化；本 crate 提供稳定的 `Prediction { logits, class_ids() }`。
- semantic 不做 NMS、不产生 box，是纯稠密逐像素分类。
- 训练时语义分割的未选像素会被标记为 ignore（不进 loss），见 [训练.md](../训练.md)。
