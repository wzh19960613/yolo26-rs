# 实例分割训练（train-seg）

用 `train` feature 在 Rust 侧训练标准 YOLO26 实例分割模型(非 YOLOE)。基于 Candle autograd,无 Python/PyTorch 依赖。

完整 API 参考见 [训练.md](../训练.md)。YOLOE 训练见 [YOLOE训练.md](YOLOE训练.md)。

## 何时用

- 在自定义数据集上训练 YOLO26 分割模型(固定类别,非开放词表)。
- 需要 Rust 原生训练循环,不依赖 Python/PyTorch。
- 想用库 API 精细控制 optimizer、freeze、lr schedule。

## 准备

- **Feature**:`--features train`(自动拉入 `segment` + `image`)。
- **数据**:Ultralytics YAML 格式的分割数据集(如 `coco8-seg/dataset.yaml`)。
- **权重**(可选):优先从预训练 `.pt` 初始化;`.safetensors` 仍可选。也可从头训练(`Model::new`)。

## Rust 示例（一步训练）

```rust
use candle_core::{DType, Device};
use yolo26_rs::{ImageSize, Scale, segment};
use yolo26_rs::train::{
    ModelConfig, OptimizerConfig, Target,
    dataset::{Dataset, Split, collate::collate_segmentation_samples, ultralytics},
};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    // 1. 构造可训练分割模型(从预训练权重初始化)
    let yaml = ultralytics::Yaml::from_file("datasets/coco8-seg/dataset.yaml")?;
    let config = ModelConfig::Segment(
        segment::config_builder()
            .with_scale(Scale::N)
            .with_labels_count(yaml.names.len())
            .build(),
    );
    let mut model = yolo26_rs::train::Model::from_pt_file("yolo26n-seg.pt", config)?;
    model.set_class_names(yaml.names.clone())?;

    // 2. 创建训练 session(绑定模型和优化器)
    let mut session = yolo26_rs::train::Session::new(
        model,
        OptimizerConfig::Sgd { learning_rate: 1e-3 },
    )?;

    // 3. 加载一个 batch 的数据
    let dataset = ultralytics::seg::from_file(
        "datasets/coco8-seg/dataset.yaml", Split::Train,
        ImageSize::square(640), ImageSize::square(160),
        DType::F32, Device::Cpu, 100,
    )?;
    let samples = (0..dataset.len().min(4))
        .map(|i| dataset.sample(i))
        .collect::<Result<Vec<_>, _>>()?;
    let batch = collate_segmentation_samples(&samples)?;

    // 4. 一步训练(input + target → loss → 反向 → 更新)
    let report = session.train_batch(&batch.input, &batch.target)?;
    println!("loss = {:.4}", report.loss);

    // 5. 保存
    session.model().save_pt("trained.pt")?;
    Ok(())
}
```

`collate_segmentation_samples` 返回的 `batch.target` 已经是 `Target::Segmentation(SegmentationTargets)`,可直接传给 `train_batch`。

## 对应官方 Python

```python
from ultralytics import YOLO

model = YOLO("yolo26n-seg.pt")
model.train(data="coco8-seg.yaml", epochs=100, batch=4, imgsz=640)
```

## API 速查

### 模型构造

| API | 说明 |
| --- | --- |
| `train::Model::new(config)` | 从空 VarMap 构造(从头训练)。 |
| `train::Model::new_with_class_names(config, names)` | 从空 VarMap 构造，并把类名写入后续 `.pt` 导出。 |
| `train::Model::from_pt_file(path, config)` | 从官方 `.pt` checkpoint 初始化(需 `pt` feature)。 |
| `train::Model::from_safetensors_file(path, config)` | 从 `.safetensors` 预训练权重初始化。 |
| `ModelConfig::Segment(segment::config_builder()...build())` | 分割任务配置。 |

### 训练 session

| API | 说明 |
| --- | --- |
| `train::Session::new(model, optimizer)` | 创建训练 session,绑定模型和优化器。 |
| `Session::new_with_variable_filter(model, optimizer, filter)` | 只训练匹配变量(实现 freeze recipe)。 |
| `Session::train_batch(&input, &target)` | 一步训练:`input`(图像张量) + `target`(`Target::Segmentation`) → loss → 反向 → 更新。 |
| `Session::train_batch_with_loss_config(&input, &target, loss_config)` | 自定义检测 loss gains。 |
| `Session::model()` | 借用底层 `train::Model`,用于保存权重。 |

### 优化器

| API | 说明 |
| --- | --- |
| `OptimizerConfig::Sgd { learning_rate }` | SGD(含 momentum)。 |
| `OptimizerConfig::AdamW { params }` | AdamW。 |
| `OptimizerConfig::MuSgd { params, learning_rate }` | MuSGD(momentum + weight decay)。 |

### 数据与 target

| API | 说明 |
| --- | --- |
| `ultralytics::seg::from_file(yaml, split, img_size, mask_size, dtype, device, cache)` | 加载 Ultralytics 分割数据集。 |
| `Dataset::sample(i)` | 取第 i 个样本(`Sample`)。 |
| `collate_segmentation_samples(&samples)` | 把多个 `Sample` 拼成一个 batch `Sample`(`input` + `Target::Segmentation`)。 |
| `Target::Segmentation(SegmentationTargets)` | 分割 target(boxes + class ids + masks)。 |

### 保存

| API | 说明 |
| --- | --- |
| `Model::save_pt(path)` | 保存为官方 `.pt` checkpoint(需 `pt` feature)，会同步当前 `labels_count`、head tensor shape 和已设置的 class names。 |
| `Model::save_safetensors(path)` | 保存为 `.safetensors`(推理侧 `segment::Model::from_file` 可直接加载)。 |

## 与官方的差异 / 注意事项

- 官方用 PyTorch Trainer + Ultralytics engine;本 crate 用 `train::Model` + `train::Session` 表达可训练网络和训练循环,无 Python 依赖。
- 训练构造的 VarMap dtype 是显式参数(`DType::F32` 在 CPU 上是自然选择),与推理侧的 `Auto` dtype 规则无关——后者只管加载已有 checkpoint。
- 当前不复刻官方 DDP/AMP/callback 和 PyTorch scheduler 对象恢复;native validation 已记录 loss、box mAP、mask mAP。
- 冻结参数用 `Session::new_with_variable_filter` 按变量名过滤实现(对应官方 `freeze` 参数)。
