# 使用场景指南索引

本目录是 `yolo26-rs` 的**按使用场景组织**的中文文档：每篇对应一种典型用法，包含可运行的 Rust 示例、对应的官方 Python 代码、所需 feature 以及涉及的 API/配置项详解。

与上级 `doc/中文/` 下按模块组织的 API 参考（`任务推理API.md`/`YOLOE接口.md`/`训练.md`）互补：本目录是「端到端用法指南」，API 参考是「逐项手册」。两者相互交叉链接。

## 场景导航

### 推理（detect 默认；其余任务需对应 feature）

- [物体检测 detect](物体检测.md)：找出图像中的目标，并给出它们的位置和类别。
- [图像分类 classify](图像分类.md)：判断整张图最可能属于哪些类别。
- [实例分割 segment](实例分割.md)：识别每个目标，并分离出它占据的像素区域。
- [语义分割 semantic](语义分割.md)：为图像中的每个像素判断所属类别。
- [姿态估计 pose](姿态估计.md)：定位目标的关键点，用于描述姿态或骨架。
- [旋转框检测 obb](旋转框检测.md)：检测有方向角的目标，适合航拍、遥感等旋转目标场景。
- [SAHI 切片检测 sahi](SAHI切片检测.md)：把大图切成小块检测，提高密集小目标的召回。

> 默认 feature 仅含 `detect`+`image`+`pt`。classify/segment/semantic/pose/obb 各需 `--features <task>`（见下方 Feature 速查）。

### YOLOE（需 `--features yoloe`）

- [YOLOE text prompt](YOLOE文本提示.md)：用文字指定要找的类别，进行开放词表检测或分割。
- [YOLOE visual prompt](YOLOE视觉提示.md)：用框或 mask 给出示例，让模型寻找同类目标。
- [YOLOE prompt-free / LRPC](YOLOE无提示LRPC.md)：不手动指定提示词，直接从内置开放词表中预测目标。

> `yoloe` 是聚合 feature（= `yoloe-text` + `yoloe-visual` + `yoloe-pf`）。只做 text prompt 可用 `--features yoloe-text`；visual prompt 用 `--features yoloe-visual`；prompt-free（含内置 4585 词表）用 `--features yoloe-pf`。text prompt 需要调用方显式提供 CLIP 权重和 tokenizer 路径（见 [YOLOE文本提示.md](YOLOE文本提示.md)）。

### 训练（需 `train` feature）

- [实例分割训练 train-seg](实例分割训练.md)：标准 YOLO26 分割训练循环。
- [YOLOE 训练 train-yoloe](YOLOE训练.md)：YOLOE text/visual/prompt-free 训练，fine-tune/from-scratch 等。

## Feature 速查

| 能力 | Feature | 何时需要 |
| --- | --- | --- |
| 检测推理 / SAHI / 读图 / 加载 `.pt` | 默认（`detect`+`image`+`pt`） | 开箱即用 |
| 分类推理 | `--features classify` | |
| 实例分割推理 | `--features segment` | |
| 语义分割推理 | `--features semantic` | |
| 姿态推理 | `--features pose` | |
| OBB 推理 | `--features obb` | |
| YOLOE 推理 | `--features yoloe` | |
| 训练（含全部任务 + YOLOE） | `--features train` | 自动拉入所有任务 + `image` |
| Apple GPU | `--features metal` | macOS Metal 后端 |
| NVIDIA GPU | `--features cuda` | 需 CUDA toolkit + nvcc |
| 浏览器 | `--features wasm` | wasm32 目标 |

> 无 CUDA toolkit 时不要用 `--all-features`，`cuda` feature 会要求 `nvcc`。默认验证命令是 `cargo build --features train`。

## 最小快速开始（detect）

```rust
use yolo26_rs::{FilterOption, Image, detect};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let image = Image::from_file("examples/bus.jpg")?;
    let model = detect::Model::from_file("yolo26s.pt")?;
    let detections = model.predict(&image, &FilterOption::default())?;
    Ok(())
}
```
