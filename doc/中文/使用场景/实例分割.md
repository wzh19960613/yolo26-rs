# 实例分割推理（segment）

本篇演示如何用 `segment` 任务根做 YOLO26 实例分割：每张图返回若干实例，每个实例带一个检测框和一个二值 mask。`predict()` 比 detect 多一个 `MaskOption`，控制 mask 是否上采样到源图分辨率。

完整 API 参考见 [任务推理API.md](../任务推理API.md)。

## 何时用

- 需要像素级实例轮廓（人、车、细胞……），不只是框。
- 需要把 mask 叠回原图做可视化或再喂给下游（OCR、测量）。
- 想要和 detect 一致的检测结果，外加 proto mask 解码。

## 准备

- **Feature**：需 `--features segment`（不在默认 feature 里）。
- **权重**：分割官方 `.pt` 权重名形如 `yolo26s-seg.pt`（`-seg` 后缀）。

```bash
cargo build --release
```

## Rust 示例

```rust
use yolo26_rs::{FilterOption, Image, MaskOption, segment};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let image = Image::from_file("examples/bus.jpg")?;
    let model = segment::Model::from_file("yolo26s-seg.pt")?;
    let mask_opt = MaskOption { high_resolution: true };
    let segs = model.predict(&image, &FilterOption::default(), &mask_opt)?;
    for s in &segs {
        let d = &s.detection;
        let mask = &s.mask;
        println!(
            "class={} conf={:.2} bbox={:?} mask={}x{}",
            d.class_id, d.confidence, d.bbox,
            mask.width, mask.height,
        );
        let _ = mask.data();
    }
    Ok(())
}
```

## 对应官方 Python 代码

```python
from ultralytics import YOLO

model = YOLO("yolo26s-seg.pt")
results = model.predict("examples/bus.jpg", imgsz=640, conf=0.25)
for r in results:
    for i, b in enumerate(r.boxes):
        m = r.masks[i]
        print(b.cls.item(), b.conf.item(), b.xyxy[0].tolist(), m.data.shape)
```

## API 与配置项详解

`segment::config_builder()` 与 detect 完全相同（默认 640、COCO 80 类），builder 方法见 [物体检测.md](物体检测.md)。

### 模型与预测

| API | 说明 |
| --- | --- |
| `segment::Model::from_file(path)` | 自动识别 `.pt` / `.safetensors` 并加载分割模型。 |
| `segment::Model::from_pt_file(path, config)` | 从官方 `.pt` 加载。 |
| `segment::Model::from_safetensors(weights, config)` | 从 `.safetensors` 字节加载。 |
| `model.forward_tensor(&input)` | raw forward，返回 `(predictions, proto)`。 |
| `model.predict(&image, &filter, &mask)` | 端到端推理，返回 `Vec<segment::Prediction>`。 |

### `MaskOption`

| 字段 | 默认 | 说明 |
| --- | --- | --- |
| `high_resolution` | `false` | `true` 时 mask 上采样到源图分辨率；`false` 时保留模型原生低分辨率 mask。 |

### `Mask`（`segment::Prediction.mask`）

| API | 说明 |
| --- | --- |
| `mask.width` / `mask.height` (`u16`) | mask 尺寸。 |
| `mask.logits: Vec<f32>` | 逐像素 logit；`>0` 表示属于实例。 |
| `mask.data()` | 返回 `Vec<u8>` 二值 mask（logit>0 为 1）。 |
| `mask.get(x, y)` | 单点是否属于实例。 |
| `mask.resize_checked(w, h)` | 双线性插值并阈值化，返回 `Result<Mask>`（推荐）。 |
| `mask.resize(w, h)`（`#[deprecated]`） | panic 版，新代码勿用。 |

### `segment::Prediction` 字段

- `detection: detect::Prediction`（含 `bbox`/`confidence`/`class_id`）。
- `mask: Mask`。

## 与官方的差异 / 注意事项

- 官方 mask 默认就在原图分辨率；本 crate 默认返回模型原生低分辨率 mask，需显式 `MaskOption { high_resolution: true }` 才上采样。
- 官方 `masks.data` 是 tensor；本 crate `Mask` 存 `Vec<f32>` logits，提供 `data()` 取二值。
- 框和 mask 都在源图坐标系，`Prediction::translated(dx,dy)` 可整体平移（切片检测复用）。
- 训练时 segment target mask 的编码（overlap/per-instance）由 `train` 模块控制，见 [训练.md](../训练.md)。
