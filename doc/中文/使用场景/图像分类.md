# 图像分类推理（classify）

本篇演示如何用 `classify` 任务根做 YOLO26 分类推理（ImageNet 风格的整图分类）。classify 与检测类任务的差异在于：输入尺寸默认 224、没有 box、`predict()` 不接受 `FilterOption`，直接返回按置信度排序的 `Vec<Prediction>`。

完整 API 参考见 [任务推理API.md](../任务推理API.md)。

## 何时用

- 整图分类（一张图一个标签），例如产品质检、场景识别。
- 需要无检测框、无 NMS 的轻量推理结果。
- 在 Rust 侧得到 top-k 概率分布自行后处理。

## 准备

- **Feature**：需 `--features classify`（不在默认 feature 里）。GPU 后端再加 `--features metal`/`--features cuda`。
- **权重**：分类官方 `.pt` 权重名形如 `yolo26s-cls.pt`（注意 `-cls` 后缀）。

```bash
cargo build --release
```

## Rust 示例

```rust
use yolo26_rs::{Image, classify};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let image = Image::from_file("examples/bus.jpg")?;
    let model = classify::Model::from_file("yolo26s-cls.pt")?;
    let predictions = model.predict(&image)?;
    for c in predictions.iter().take(5) {
        println!("class={} conf={:.2}", c.class_id, c.confidence);
    }
    Ok(())
}
```

## 对应官方 Python 代码

```python
from ultralytics import YOLO

model = YOLO("yolo26s-cls.pt")
results = model.predict("examples/bus.jpg", imgsz=224)
probs = results[0].probs
for i in probs.top5:
    print(i, probs.data[i].item())
```

## API 与配置项详解

### `classify::config_builder()`（返回 `model::config::base::Builder`）

classify 的默认 builder 把输入尺寸设为 **224**（不是 640），类别数默认 **ImageNet(1000)**。其余 builder 方法与 detect 完全一致（`with_scale`/`with_device`/`with_dtype`/`with_input_size`/`with_image_size`/`with_labels_count` 等，详见 [物体检测.md](物体检测.md)）。

| builder 方法 | 默认 | 说明 |
| --- | --- | --- |
| `with_input_size(n)` | `224` | 分类默认 224；改自定义尺寸会被对齐到 32。 |
| `with_labels_count(n)` | `IMAGENET.len()` | 自定义分类数据集时改这里。 |

### 模型与预测

| API | 说明 |
| --- | --- |
| `classify::Model::from_file(path)` | 自动识别 `.pt` / `.safetensors` 并加载分类模型。 |
| `classify::Model::from_pt_file(path, config)` | 从官方 `.pt` 加载。 |
| `classify::Model::from_safetensors(weights, config)` | 从 `.safetensors` 字节加载。 |
| `model.forward_tensor(&input)` | raw forward，返回 `[B, nc]`。 |
| `model.predict(&image)` | 端到端推理，返回按置信度降序的 `Vec<classify::Prediction>`。 |

### `classify::Prediction` 字段

- `class_id: u32`。
- `confidence: f32`（`[0,1]`）。

> classify 没有 `PredictOptions` 类型别名，因为它不过滤——你拿到全量结果后自己取 top-k。

## 与官方的差异 / 注意事项

- 官方 `probs` 对象带 `top1`/`top5`/`argmax` 等便利方法；本 crate 返回已排序的 `Vec`，`predictions[0]` 即 top-1。
- 官方 classify 也支持 `.classify()`/训练；本 crate 的 classify 只做推理，训练走统一的 `train` 模块（见 [训练.md](../训练.md)）。
- 类别数默认按 ImageNet 1000 类推断（`labels_count` 从 checkpoint 的 head shape 推断）；类名表 `default_labels::IMAGENET` 需启用 `default_labels` feature（默认关），或用 `.pt` 权重的 `pt_loader::load_pt_metadata().names` 直接读取。
