# 姿态 / 关键点推理（pose）

本篇演示如何用 `pose` 任务根做 YOLO26 姿态推理：每个检测到的人/对象带一个检测框和一组关键点（如 COCO 17 点骨架）。pose 用的是专用配置 `ForPose`，可自定义关键点数量和每点维度。

完整 API 参考见 [任务推理API.md](../任务推理API.md)。

## 何时用

- 人体/动物姿态估计（动作识别、运动分析、手势）。
- 需要每个实例的骨架关键点 + 可见性。
- 需要在框基础上叠加 keypoint 坐标做下游计算。

## 准备

- **Feature**：需 `--features pose`（不在默认 feature 里）。
- **权重**：姿态官方 `.pt` 权重名形如 `yolo26s-pose.pt`（`-pose` 后缀）。
- **默认**：17 个关键点，每点 3 个值（x, y, visibility）。

```bash
cargo build --release
```

## Rust 示例

```rust
use yolo26_rs::{FilterOption, Image, pose};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let image = Image::from_file("examples/bus.jpg")?;
    let model = pose::Model::from_file("yolo26s-pose.pt")?;
    let poses = model.predict(&image, &FilterOption::default())?;
    for p in &poses {
        println!(
            "class={} conf={:.2} bbox={:?} kpts={}",
            p.class_id, p.confidence, p.bbox,
            p.keypoints.len(),
        );
        for (i, k) in p.keypoints.iter().enumerate() {
            println!("  kp{i}: ({:.1},{:.1}) vis={:?}", k.x, k.y, k.visibility);
        }
    }
    Ok(())
}
```

## 对应官方 Python 代码

```python
from ultralytics import YOLO

model = YOLO("yolo26s-pose.pt")
results = model.predict("examples/bus.jpg", imgsz=640, conf=0.25)
for r in results:
    for i, b in enumerate(r.boxes):
        kpts = r.keypoints[i]
        print(b.cls.item(), b.conf.item(), b.xyxy[0].tolist(), kpts.xy.shape)
```

## API 与配置项详解

pose 用专用配置 `Config = model::config::ForPose`，内部含 `base: Base` + 关键点参数。

### `pose::config_builder()`（返回 `model::config::for_pose::Builder`）

继承所有 base builder 方法（`with_scale`/`with_device`/`with_dtype`/`with_input_size`/`with_labels_count` 等，见 [物体检测.md](物体检测.md)），并额外提供：

| builder 方法 | 默认 | 说明 |
| --- | --- | --- |
| `with_keypoints_count(n)` | `17` | 每个实例的关键点数量。 |
| `with_keypoint_dims(n)` | `3` | 每个关键点的值数（至少 2：x/y；COCO 为 3：x/y/visibility）。 |

### 模型与预测

| API | 说明 |
| --- | --- |
| `pose::Model::from_file(path)` | 自动识别 `.pt` / `.safetensors` 并加载姿态模型。 |
| `pose::Model::from_pt_file(path, config)` | 从官方 `.pt` 加载。 |
| `pose::Model::from_safetensors(weights, config)` | 从 `.safetensors` 字节加载。 |
| `model.forward_tensor(&input)` | raw forward，返回 `[B, A, nc+5+nk*kd]`。 |
| `model.predict(&image, &filter)` | 端到端推理，返回 `Vec<pose::Prediction>`。 |

### `pose::Prediction` 字段

- `bbox: BBox`、`confidence: f32`、`class_id: u32`（与 detect 一致）。
- `keypoints: Vec<Keypoint>`。

### `Keypoint` 字段

- `x: f32`、`y: f32`（源图坐标）。
- `visibility: Option<f32>`（关键点可见性/置信度）。

## 与官方的差异 / 注意事项

- 官方 `keypoints` 对象提供 `xy`/`xyn`/`conf` 等 tensor 视图；本 crate 返回 `Vec<Keypoint>` 强类型，`visibility` 单独可选。
- 关键点骨架连接（如 COCO 17 点的骨骼）需要你自行维护。
- 自定义关键点数据集（如动物 17 点、手 21 点）时务必 `with_keypoints_count`/`with_keypoint_dims` 与权重匹配。
