# 物体检测推理（detect）

本篇演示如何用 `yolo26-rs` 的 `detect` 任务根做 YOLO26 物体检测推理：加载官方 `.pt` 权重、跑一张图、按置信度和类别过滤结果。六个推理任务根共享同一套形状，detect 是其中最基础的一个，可作为其它任务的模板。

完整 API 参考见 [任务推理API.md](../任务推理API.md)。

## 何时用

- 需要在 Rust 服务、WASM、嵌入式或无 Python 环境里跑 YOLO26 检测。
- 需要强类型、轻量的检测结果（`BBox`/`confidence`/`class_id`），自己控制可视化、保存或后处理。
- 需要在 CPU、Metal 或 CUDA 上本地推理（`device::auto()` 自动选择）。

## 准备

- **Feature**：detect 是默认能力，**无需额外 feature**。要使用 GPU 时编译加 `--features metal`（Apple）或 `--features cuda`（NVIDIA，需 nvcc）。
- **权重**：官方 `.pt`（默认 `pt` feature），如 `yolo26s.pt`，直接用 `from_pt_file` 加载；也可用转换后的 `.safetensors`（`from_safetensors`）。
- **依赖**：示例用 `Image::from_file` 一行读图，需启用 `image` feature（默认已含在 `default` 里）。不启用时自行用任意解码器构造 `Image::new(...)`。

```bash
cargo build --release
cargo build --release --features metal
```

## Rust 示例

```rust
use yolo26_rs::{FilterOption, Image, detect};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let image = Image::from_file("examples/bus.jpg")?;
    let model = detect::Model::from_file("yolo26s.pt")?;
    let detections = model.predict(&image, &FilterOption::default())?;
    for d in &detections {
        println!(
            "class={} conf={:.2} bbox={:?}",
            d.class_id, d.confidence, d.bbox,
        );
    }
    Ok(())
}
```

只保留 person 和 bus（COCO class id 0 和 5），用 `FilterOption`：

```rust
use yolo26_rs::{FilterOption, Image, detect};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let image = Image::from_file("examples/bus.jpg")?;
    let model = detect::Model::from_file("yolo26s.pt")?;
    let filter = FilterOption {
        confidence_threshold: 0.25,
        class_filter: vec![0, 5],
    };
    let detections = model.predict(&image, &filter)?;
    Ok(())
}
```

## 对应官方 Python 代码

```python
from ultralytics import YOLO

model = YOLO("yolo26s.pt")
results = model.predict(
    "examples/bus.jpg",
    imgsz=640,
    conf=0.25,
    classes=[0, 5],
)
for r in results:
    for b in r.boxes:
        print(b.cls.item(), b.conf.item(), b.xyxy[0].tolist())
```

## API 与配置项详解

### `detect::config_builder()`（返回 `model::config::base::Builder`）

| builder 方法 | 类型/默认 | 说明 |
| --- | --- | --- |
| `with_scale(s)` | `Scale::{N,S,M,L,X}`，默认 `N` | 对应权重文件名中的 `n/s/m/l/x`。 |
| `with_device(d)` | `Device`，默认 `Device::Cpu` | 用 `device::auto()` 自动选 GPU，或 `DeviceSpec::Metal(0).to_device()`。 |
| `with_dtype(t)` | `DType`，默认 **Auto** | 强制运行时精度（如 `F16`/`F32`），覆盖 Auto。**不调用时默认 `DtypeRequest::Auto`**：综合 device + target arch + 权重 dtype 解析——GPU（CUDA/Metal）上与权重一致（F16 权重 → F16），CPU / wasm32 上强制 F32（即使权重是 F16）。因此 CPU 部署无需手写 `with_dtype(F32)`。 |
| `with_input_size(n)` | `usize`，默认 `MODEL_INPUT_SIZE`(640) | 方形输入，会被对齐到 32 的倍数。 |
| `with_image_size(w,h)` | `usize, usize` | 非方形 `(width,height)` 输入，同样对齐到 32。 |
| `with_max_predictions(n)` | `usize`，默认 `300` | head 保留的最大预测数。 |
| `with_rectangular_padding(b)` | `bool`，默认 `true` | 预处理是否保留矩形（不做正方形 letterbox）。 |
| `with_labels_count(n)` | `usize`，默认 COCO(80) | 类别数，自定义数据集时改这里。 |

### 模型与预测

| API | 说明 |
| --- | --- |
| `detect::Model::from_file(path)` | 自动识别 `.pt` / `.safetensors` 并加载。 |
| `detect::Model::from_pt_file(path, config)` | 直接从官方 `.pt` 加载（经 `pt_loader`）。 |
| `detect::Model::from_safetensors(weights, config)` | 从 `.safetensors` 字节加载。 |
| `model.forward_tensor(&input)` | 对预处理 tensor 做 raw forward，返回 `[B, A, nc+4+1]`。 |
| `model.predict(&image, &filter)` | 端到端推理，返回 `Vec<detect::Prediction>`。 |

### `FilterOption`（`detect::PredictOptions` 的别名）

| 字段 | 默认 | 说明 |
| --- | --- | --- |
| `confidence_threshold` | `0.25` | 低于此分数的检测被丢弃。 |
| `class_filter` | `vec![]` | 空表示全部保留；否则只保留列出的 class id。 |

### `detect::Prediction` 字段

- `bbox: BBox`（`x_min/y_min/x_max/y_max`，源图坐标）。
- `confidence: f32`（`[0,1]`）。
- `class_id: u32`。

## 与官方的差异 / 注意事项

- 官方 `YOLO("model.pt")` 直接吃 `.pt`；本 crate 同样以 `.pt` 为主（`from_pt_file`，默认 `pt` feature），`from_safetensors` 作为可选字节加载路径。
- 官方能吃路径/URL/numpy/PIL；本 crate 高级 API 用统一 `Image`，提供 `Image::from_file`（`image` feature 下）一行读图，或自行构造 `Image::new`。
- 官方结果对象带可视化/保存/DataFrame；本 crate 返回轻量 Rust 结构。
- GPU 编译：无 CUDA toolkit 时不要用 `--all-features`，`cuda` feature 会要求 `nvcc`。详见 [0.索引.md](../0.索引.md) 的快速开始。
