# 训练

> **训练能力尚未经过严格测试，请谨慎使用；当前不保证训练效果、收敛质量或与官方 PyTorch Trainer 的数值一致性。建议先用小数据集和短训练验证完整流程，再投入正式训练。**

训练能力位于 `train` feature 下，基于 Candle autograd、`VarMap` 和 optimizer。它提供 Rust-native 的 YOLO26 训练基础设施，而不是直接封装 Ultralytics Python Trainer；非 YOLO26 家族的训练器或参数不属于本 crate 的实现范围。

训练依赖 Candle autograd 和后端卷积实现；Candle 0.10.2 的两个路径可能影响梯度正确性：

- `upsample_nearest1d/2d` 的 backward 在已有梯度存在时应累加卷积求和结果，而不是覆盖已有梯度。
- Metal `conv1d/conv2d` 路径在 matmul 前应把可能非连续的 kernel materialize 成连续 buffer，并使用零偏移 contiguous layout，避免卷积梯度路径遇到非连续 kernel 时读错偏移。

需要训练，尤其是 Metal 训练时，请先按 [Candle 0.10.2 Patch Guide](Candle-0.10.2补丁指南.md) 使用本地 patched `candle-core`，或等待包含修复的上游版本。

端到端用法指南:[标准分割训练](使用场景/实例分割训练.md) · [YOLOE 训练](使用场景/YOLOE训练.md)。

## 快速开始

```rust
use yolo26_rs::{Scale, detect, train};

let yaml = train::dataset::ultralytics::Yaml::from_file("data.yaml")?;
let config = train::ModelConfig::Detect(
    detect::config_builder()
        .with_scale(Scale::S)
        .with_labels_count(yaml.names.len())
        .build(),
);
let model = train::Model::new_with_class_names(config, yaml.names.clone())?;
let mut session = train::Session::new(
    model,
    train::OptimizerConfig::Sgd { learning_rate: 1e-4 },
)?;

let report = session.train_batch(&input, &target)?;
let eval = session.eval_batch(&input, &target)?;
```

分类目录数据也可以直接作为数据集：

```text
dataset/
  train/cat/*.jpg
  train/dog/*.jpg
  val/cat/*.jpg
  val/dog/*.jpg
```

## 完整 API

### 核心类型

| API | 说明 |
| --- | --- |
| `train::Task` | `Detect`、`Classify`、`Segment`、`Pose`、`Semantic`、`Obb`。 |
| `train::ModelConfig` | 各任务配置的枚举封装。 |
| `train::Model::new(config)` | 新建可训练模型和变量。 |
| `train::Model::new_with_class_names(config, names)` | 新建可训练模型，并在后续 `.pt` 导出中写入官方 `model.names`。`names.len()` 必须等于 `labels_count`。 |
| `Model::set_class_names(names)` | 为已构造模型设置 `.pt` 导出类名。 |
| `Model::from_pt_file(path, config)` | 用官方 `.pt` 初始化可训练模型。 |
| `Model::load_pt_file(path)` | 从官方 `.pt` 加载匹配变量并返回 `LoadReport`。 |
| `Model::save_pt(path)` | 保存为官方 `.pt`，并写入当前 class metadata。 |
| `Model::save_pt_with_names(path, names)` | 保存为官方 `.pt` 并显式写入类名。 |
| `Model::from_safetensors(weights, config)` | 用 safetensors 初始化可训练模型（可选兼容路径）。 |
| `Model::load_safetensors(weights)` | 从 safetensors 加载匹配变量并返回 `LoadReport`。 |
| `Model::save_safetensors(path)` | 保存当前训练变量为 safetensors。 |
| `Model::forward_raw(input)` | 训练模式 raw forward。 |
| `Model::forward_raw_eval(input)` | eval 模式 raw forward。 |
| `train::Output::DetectE2e` | 普通 detection 训练 raw output；包含 one-to-many 和 one-to-one 两路检测 head。eval raw output 和推理仍使用 one-to-one。 |
| `train::Output::SegmentE2e` | instance segmentation 训练 raw output；包含 one-to-many 和 one-to-one 两路 detect/mask head，并共享 proto。eval raw output 和推理仍使用 one-to-one。 |
| `train::Output::PoseE2e` | pose 训练 raw output；包含 one-to-many 和 one-to-one 两路 detect/keypoint head。eval raw output 和推理仍使用 one-to-one。 |
| `train::Output::ObbE2e` | OBB 训练 raw output；包含 one-to-many 和 one-to-one 两路 detect/angle head。eval raw output 和推理仍使用 one-to-one。 |
| `train::Session` | 训练/评估会话。 |
| `Session::new(model, optimizer)` | 训练全部变量。 |
| `Session::new_with_variable_filter(...)` | 只训练匹配变量，例如 head-only。 |
| `Session::save_optimizer_state_safetensors(path)` | 保存 AdamW/MuSGD optimizer 内部状态；SGD 没有内部状态时返回 `false`。 |
| `Session::load_optimizer_state_safetensors(path)` | 从 safetensors sidecar 恢复当前 optimizer 内部状态。 |
| `train::OptimizerConfig` | `Sgd { learning_rate }`、`AdamW { params }` 或 `MuSgd { params }`；本地 `Sgd` 只有 `learning_rate`，不支持 weight decay（已知差异，AdamW/MuSGD 的 weight decay 已支持）。 |
| `train::AutoOptimizerSelection::ultralytics(classes, iterations)` | 对齐官方 `optimizer=auto` 的选择规则。短训练选择 AdamW，长训练选择 MuSGD。 |
| `train::ParamsMuSgd` | Native MuSGD 参数，包含 Muon/SGD 混合权重、momentum、nesterov 和 weight decay。 |
| `train::SampleOrder::ultralytics(seed, deterministic)` | 对齐官方 `seed` / `deterministic` 的本地样本顺序配置；非零 deterministic seed 会产生稳定索引置换。 |
| `train::DetectionLossConfig` | 检测类监督 loss 参数，包含 box/class/dfl/pose/keypoint-objectness/angle 权重、`tal_topk`、`tal_topk2`、`tal_alpha` 和 `tal_beta`；`tal_topk2` 用于官方 STAL/one-to-one 路径的二阶段候选筛选。小目标候选按官方 STAL 把过小宽/高扩到第二层 stride，分类 target score 按官方 `align_metric * pos_overlaps / pos_align_metrics` 归一化。 |
| `train::ProgressiveLossSchedule` | YOLO26 E2E loss 的 one-to-many / one-to-one 权重调度；默认从 `0.8/0.2` 衰减到 `0.1/0.9`。 |
| `train::SegmentationMaskEncoding` | Segment target mask 编码；`Overlap` 表示官方 `overlap_mask=True` 的单通道 instance-index map，`PerInstance` 表示每 object 一个二值 mask。 |
| `train::ClassFilter::new(single_class, classes)` | 对齐官方 `single_cls` / `classes` 的训练目标过滤与类 id remap；语义分割未选像素会进入 ignore loss。 |
| `train::Freeze::first_layers(n)` / `Freeze::layers([...])` | 对齐官方 `freeze`，冻结前 N 个 `model.{idx}.` 层或指定层索引，并识别官方始终冻结的 `.dfl` 变量。 |
| `train::CheckpointReport` | `RunnerReport.checkpoints` 中的 checkpoint 摘要，记录 `last.pt`、`best.pt`、best epoch/loss 和周期性 checkpoint。 |
| `train::ResumeState` | checkpoint sidecar 中的轻量训练状态，记录 completed epoch、completed step 和 best loss。 |

### 数据集适配

| API | 说明 |
| --- | --- |
| `Yaml::from_file(path)` | 解析 YOLO YAML。 |
| `detection_dataset_from_file(...)` | 检测数据集。 |
| `segmentation_dataset_from_file(...)` | 实例分割数据集。 |
| `segmentation_dataset_from_file_with_overlap_mask(...)` | 显式选择 segment target mask 编码；默认入口使用官方 `overlap_mask=True`。 |
| `pose_dataset_from_file(...)` | 姿态数据集，读取 `kpt_shape`。 |
| `semantic_dataset_from_file(...)` | 语义分割数据集。 |
| `obb_dataset_from_file(...)` | OBB 数据集。 |
| `ClassificationDataset::from_dir(...)` | 分类目录数据集。 |

YAML split 支持图片目录、图片列表 `.txt`、单张图片和常见 glob，包括 `**` 递归目录。

自定义类别训练时，先用 `Yaml::from_file(data_yaml)` 读取官方数据集 YAML，再把 `yaml.names.len()` 传给任务 config 的 `with_labels_count(...)`。用 `Model::new_with_class_names(config, yaml.names.clone())` 或 `Model::set_class_names(...)` 设置类名后，`Model::save_pt("best.pt")` 会同时改写 `.pt` 中的 head tensor metadata、`model.nc` 和 `model.names`；官方 Python `YOLO("best.pt")` 可直接读取这些类名。若没有设置类名，导出 `.pt` 时会按类别数生成 `class_0`、`class_1` 等占位名，避免 `nc` 和 `names` 不一致。

### 训练和评估

`RunnerConfig` 控制 `epochs`、`batch_size`、`steps_per_epoch`、`accumulate_steps`、`sample_fraction`、`sample_order`、`time_limit_hours`、`loss_config`、`class_filter`、`early_stopping`、`resume_state`、`learning_rate_schedule`、`learning_rate_warmup`、`bias_learning_rate_warmup`、`momentum_warmup`、`log_every_steps`、`checkpoint_dir`、`checkpoint_every_steps` 和 `checkpoint_every_epochs`。当设置 `checkpoint_dir` 时，训练 loop 会写出官方风格的 `last.pt` 和 `best.pt`，并为每个 checkpoint 写 `*.train-state.json` 与 `*.optimizer.safetensors` sidecar；默认 `train_dataset()` 按 epoch mean training loss 选择 `best`，`train_dataset_with_epoch_fitness()` 可传入每 epoch validation fitness 来按官方方向选择 `best`。`loss_config` 对齐官方 detection-style loss gain，默认 `box=7.5`、`cls=0.5`、`dfl=1.5`、`pose=12.0`、`kobj=1.0`、`angle=1.0`。`sample_order` 来自 `SampleOrder::ultralytics(seed, deterministic)`，用于本地 train/eval 样本索引顺序。`time_limit_hours` 对齐官方 `time` 的小时单位训练时长限制，触发后 `RunnerReport` 会记录 `elapsed_seconds` 和 `time_limit_reached`。`EvalLoopConfig` 控制 `batch_size`、`steps`、`max_detections`、`confidence_threshold`、`iou_threshold`、`sample_order`、`loss_config` 和 `class_filter`。

### 训练参数（对齐官方语义）

库 API 通过上述 config 类型表达官方训练参数语义。常用的官方参数及其库映射：

| 官方参数 | 库 API | 说明 |
| --- | --- | --- |
| `epochs` / `batch` / `imgsz` | `RunnerConfig` | epoch 数、batch 大小、输入尺寸。 |
| `lr0` / `lrf` / `cos_lr` | `RunnerConfig` 学习率调度 | 初始 LR、final LR fraction、cosine one-cycle。 |
| `optimizer` | `OptimizerConfig` / `AutoOptimizerSelection` | `auto`/`sgd`/`adamw`/`musgd`。 |
| `box` / `cls` / `dfl` / `pose` / `kobj` / `angle` | `DetectionLossConfig` | loss gain，默认 `7.5/0.5/1.5/12.0/1.0/1.0`。 |
| `augment` / HSV / flip / mosaic / mixup | `AugmentConfig` + `AugmentingDataset` | 增强主开关默认关，开启用官方默认。 |
| `freeze` | `Freeze` | 冻结前 N 层或指定层索引。 |
| `single_cls` / `classes` | `ClassFilter` | 训练目标过滤与类 id remap。 |
| `close_mosaic` | `AugmentingDataset` 共享 epoch 计数器 | 末 N epoch 关 mosaic/mixup。 |
| `ema` | `ModelEma`（`--ema-decay` 等价的 builder 字段） | shadow 张量 lerp。 |
| `patience` | `EarlyStopping` | early-stop patience。 |
| `nbs` / `weight_decay` | `RunnerConfig` accumulate + optimizer 参数 | nominal batch size、weight decay 缩放。 |
| `seed` / `deterministic` | `SampleOrder::ultralytics` | 样本顺序。 |
| `time` | `RunnerConfig::time_limit_hours` | 训练时长限制（小时）。 |
| `mask_ratio` / `overlap_mask` | `segmentation_dataset_from_file*` | segment target mask 尺寸与 overlap 编码。 |

## YOLOE 训练

YOLOE 训练在通用训练类型之上增加 prompt 对齐。**完整示例和两种"模式"概念区分见 [使用场景/YOLOE训练.md](使用场景/YOLOE训练.md)。** 这里列 API 速查。

YOLOE 训练有两个独立的模式枚举,不要混淆:

- **`PromptMode`**(`yoloe::segment::model::train_config::PromptMode`):构造网络时选定的 prompt 对齐方式,取值 `TextPrompt` / `Visual` / `PromptFree`。传给 `ModelConfig::new`。
- **`train::yoloe::Mode`**:对齐官方 trainer recipe,决定冻结哪些参数。取值 `FineTune` / `LinearProbe` / `FromScratch` / `Visual` / `PromptFree`。

| API | 说明 |
| --- | --- |
| `ModelConfig::new(scale, device, dtype, prompt_mode)` | 构造可训练 YOLOE seg 配置,`prompt_mode` 选定网络层的 prompt 对齐方式。 |
| `Model::new(config)` / `Model::from_pt_file(config, path)` / `Model::from_safetensors(config, path)` | 新建/加载可训练模型，`.pt` 为优先初始化路径。 |
| `train::yoloe::Session::new(model, optimizer)` | YOLOE 训练 session,绑定模型和优化器。 |
| `Session::new_with_variable_filter(model, optimizer, filter)` | 只训练匹配变量(实现 freeze recipe)。 |
| `Session::text_batch(&input, &target, &prompts)` | text-prompt 一步训练,`prompts` 是 CLIP 编码的 `EmbeddingTable`。 |
| `Session::visual_batch(&input, &target, &visuals, classes)` | visual-prompt 一步训练,`visuals` 是 `BatchVisuals`(box/mask)。 |
| `Session::prompt_free_batch(&input, &target)` | prompt-free 一步训练,无 prompt 参数。 |
| `train::yoloe::Mode` | 官方 trainer recipe:`FineTune`/`LinearProbe`(→ `YOLOEPESegTrainer`)、`FromScratch`(→ `YOLOESegTrainerFromScratch`)、`Visual`(→ `YOLOESegVPTrainer`)、`PromptFree`(→ `YOLOEPEFreeTrainer`)。 |
| `Mode::official_segment_trainer()` | 返回对应的官方 trainer 名称。 |
| `Session::save_pt(path)` / `save_safetensors(path)` | 保存训练后的权重，`.pt` 为优先输出格式。 |

## 与官方的差异

- 官方 `model.train(...)` 是 PyTorch Trainer，包含完整 augmentation、scheduler、validator、AMP/DDP、resume、callback 生态；本 crate 是 Candle-native 训练基础设施。
- 官方 YOLOE Trainer 有 `YOLOEPETrainer`、`YOLOEPESegTrainer`、`YOLOESegTrainerFromScratch`、`YOLOESegVPTrainer`、`YOLOEPEFreeTrainer` 等类；本 crate 通过 `Session::train_config()` 表达对应的 trainer、官方 freeze recipe、grounding 数据需求和 `single_cls` 语义。库层和示例都使用 `train::yoloe::Session` batch API 覆盖 text、visual、prompt-free 三种 train/eval step。
- 训练 checkpoint 路径以 `.pt` 结尾时走 `pt_loader::save_pt` / `Model::save_pt` / `Seg::save_pt` 模板法写回官方 PyTorch 可读 `.pt`（`pt` feature，默认；嵌入 40 个 `(task, scale)` 官方 `data.pkl` 模板，固实压缩 ~950 KiB，6 标准任务 + YOLOE segment 三种 mode 均支持）。标准任务导出会按当前模型 tensor shape 重写 `data.pkl`，支持任意 `labels_count`；设置 class names 时会同步写入官方 `model.names`。EMA/optimizer/`proto.semseg` 等非模型 storage 与 `num_batches_tracked` 用零填充。`.safetensors` 仍可选（含 `.train-state.json` + `.optimizer.safetensors` sidecar）。
- YOLOE text-prompt 训练的类名 embedding 用 `yoloe-text` feature 下的 CLIP 编码器（`mobileclip2-rs` 依赖）生成 `[classes, 512]` embedding;prompt-free 训练不需要 prompt table,直接用 checkpoint 内部 LRPC 输出。详见 [使用场景/YOLOE训练.md](使用场景/YOLOE训练.md)。
- 当前不复刻官方 DDP/AMP/callback、PyTorch scheduler 对象恢复和完整多数据集 grounding pipeline；native validation 已记录 loss、box mAP、mask mAP。详见下方训练 loss 对齐与差异说明。
